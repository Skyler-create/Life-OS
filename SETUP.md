# LifeOS — Set up your own copy

**If you're a human:** give this entire file to Claude (claude.ai, the
desktop app, or Claude Code) and say "set up LifeOS for me." Claude will do
almost everything. Total cost: $0/month for the core system (free Supabase +
free GitHub Pages + your existing Claude subscription for the daily brief).
The optional in-app chat costs ~$2–6/month via your own Anthropic API key.

**If you're Claude:** you are setting up LifeOS — a personal task/energy/
streak system — for the person you're talking to. Follow the steps in order.
Where you have tools (Supabase MCP, shell, browser), do the work yourself;
where you don't, give the human exact click-by-click instructions and wait.
Do not skip the interview. Do not invent values.

---

## What you're building

- A mobile-first web app (single index.html, in this repo) on GitHub Pages:
  tasks with energy tags (DEEP/FOCUS/LIGHT/PARALLEL), auto-chunking, daily
  plan, evening reflection with a weighted score and streaks, calendar,
  trends, flashcards.
- A free Supabase project as the database (schema in `template/schema.sql`).
- A daily "morning brief" generated by a Claude Routine
  (claude.ai/code/routines — runs on the user's Claude subscription, in the
  cloud, laptop-free).
- Optional: in-app AI chat (edge function, user's API key) and weekly
  sweeper/export routines.

## Step 0 — Interview the user (5 questions, keep it short)

1. Email address (this becomes the single account — the whole system is
   locked to it).
2. Their schedule shape: work/school hours, fixed commitments, when their
   peak-focus hours are, evening cutoff, sleep target. (This personalizes
   the brief — it replaces the [[ ... ]] blocks in the routine templates.)
3. Weekly recurring goals (e.g. gym 2x, reading 3x) with rough minutes each.
4. Do they want the daily learning topic + flashcards in their brief? Which
   subjects interest them?
5. Do they want the paid in-app chat (~$2–6/month, needs their Anthropic API
   key), or skip it? (Everything else works without it. They can also use
   the free option: the Claude mobile app + Supabase connector.)

## Step 1 — Database (Supabase, free)

1. Create a free account at supabase.com, then a new project (free tier).
   If you (Claude) have the Supabase MCP connector, create the project and
   run the SQL yourself; otherwise walk the human through the dashboard.
2. Take `template/schema.sql` from this repo, replace every
   `__OWNER_EMAIL__` with the user's email in lowercase, and run it in the
   SQL editor. It creates 6 tables, row-level security locked to their
   email, the chat SQL function, and live sync.
3. Seed their weekly goals, e.g.:
   insert into weekly_goals (name, target_per_week, est_minutes, energy, priority, sort)
   values ('Gym', 2, 60, 'LIGHT', 'med', 1), ('Meditation', 3, 10, 'FOCUS', 'med', 2);
4. Note the project's URL and publishable key (Project Settings → API):
   needed in Step 2.
5. Recommended: Authentication → Sign In / Providers → disable "Confirm
   email" (single-user system; removes signup friction).

## Step 2 — The app (GitHub Pages, free)

1. The user needs a GitHub account. On this repo's page, click
   **Use this template** (or Fork) → create their own PUBLIC repo.
2. In their copy, edit `index.html` — the CONFIG block at the top of the
   script (three lines: SB_URL, SB_KEY, OWNER). Set their Supabase URL,
   publishable key, and email (lowercase). The publishable key is safe to
   be public; row-level security does the guarding.
3. Repo Settings → Pages → deploy from branch → main → / (root) → Save.
4. Their app is at https://<username>.github.io/<repo-name>/ in ~2 minutes.
   (Check the repo's Deployments page if it doesn't appear — a red X means
   re-push with an empty commit.)
5. In the app: "First time? Create account" with their email → confirm →
   sign in. On the phone: open the URL → Add to Home Screen.

## Step 3 — The morning brief (Claude Routine, free with subscription)

1. Make sure the user's claude.ai account has the Supabase connector:
   Settings → Connectors → add custom connector →
   https://mcp.supabase.com/mcp?project_ref=<their-project-ref>
   (Optionally also connect Gmail and Google Calendar for inbox triage.)
2. Go to claude.ai/code/routines → new routine. Paste
   `template/routine_morning_brief.md` (below its `---`), then REPLACE:
   - `__PROJECT_REF__` with their Supabase project ref
   - the two [[ ... ]] blocks with the interview answers from Step 0
   - delete step 6 if they skipped the learning topic
3. Trigger: daily at their wake time. Enable the Supabase (and optionally
   Gmail/Calendar) connectors for the routine. Run it once manually —
   a brief should appear in the app's BRIEF tab within ~2 minutes.
4. Optional: add the sweeper and export routines from
   `template/routines_weekly.md` (export needs a private repo attached).

## Step 4 — Optional in-app chat (paid, ~$2–6/month)

Skip freely; the CHAT tab will just show an error until configured.
1. The user creates an API key at console.anthropic.com (they add billing
   or credits — their money, warn them).
2. Deploy `template/edge-chat.ts` as edge function "chat" and
   `template/edge-regen-brief.ts` as "regen-brief" in their Supabase project
   (dashboard → Edge Functions, or CLI, or MCP). Replace `__OWNER_EMAIL__`,
   `__TIMEZONE__` (IANA, e.g. America/Chicago), and the
   `__PERSONAL_CONTEXT__` block first.
3. Supabase → Edge Functions → Secrets: the user adds ANTHROPIC_API_KEY
   themselves (never paste keys into chat). For the "new brief" button:
   add an API trigger to the morning routine and save its URL + token as
   ROUTINE_TRIGGER_URL / ROUTINE_TRIGGER_TOKEN secrets.

## Step 5 — Verify (do all of these)

- App loads, sign-in works, adding a task works, checking it off works.
- Run the brief routine manually → brief appears in BRIEF tab, START HERE
  shows a task title (not an id), and:
  select jsonb_array_length(plan) from briefs where date = current_date;
  returns ≥ 1.
- REFLECT tab shows a live score once the plan exists.
- If chat was configured: "what's on my plate?" answers with their real
  tasks.
